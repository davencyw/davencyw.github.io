Efficient Attention Mechanisms

Technical Log

ML Transformers

Overview

Standard self-attention has \(O(n^2)\) memory complexity with respect to sequence length, making it prohibitive for long sequences. This log covers techniques to reduce this complexity while maintaining model quality.

The Memory Bottleneck

In standard attention, we compute:

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \]

The attention matrix \(QK^T\) has shape \((n \times n)\) where \(n\) is sequence length. For a sequence of 16K tokens with fp16, this requires ~512MB just for the attention scores.

FlashAttention

FlashAttention reformulates attention computation to avoid materializing the full \(n \times n\) matrix. Key ideas:

  • Tiling: Compute attention in blocks that fit in SRAM
  • Recomputation: Recompute attention during backward pass instead of storing
  • Kernel fusion: Fuse softmax, dropout, and matmul into single kernel

Memory Complexity Comparison

Method Memory Speed
Standard Attention \(O(n^2)\) Baseline
FlashAttention \(O(n)\) 2-4x faster
FlashAttention-2 \(O(n)\) ~2x faster than FA1

Practical Notes

  • FlashAttention requires SM80+ GPUs (A100, H100, RTX 30xx/40xx)
  • Head dimensions should be 64, 128, or 256 for optimal performance
  • Causal masking is "free" in FlashAttention
  • Custom attention patterns require FlashAttention-2 or modifications
