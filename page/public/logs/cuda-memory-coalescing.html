CUDA Memory Coalescing Patterns

Technical Log

CUDA Performance GPU

Overview

Memory coalescing is one of the most critical optimization techniques for achieving high throughput on NVIDIA GPUs. When threads in a warp access consecutive memory addresses, the hardware can combine these accesses into fewer memory transactions, dramatically improving bandwidth utilization.

The Problem

Global memory on GPUs has high latency (~400-800 cycles) and limited bandwidth. Without proper access patterns, a kernel can become memory-bound, achieving only a fraction of the theoretical peak performance.

Coalesced vs Non-Coalesced Access

Consider a simple example where each thread accesses an element from an array: 

// Coalesced access - threads access consecutive elements
__global__ void coalesced(float* data, float* out, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        out[idx] = data[idx] * 2.0f;
    }
}

// Non-coalesced access - strided pattern
__global__ void strided(float* data, float* out, int n, int stride) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    int strided_idx = idx * stride;
    if (strided_idx < n) {
        out[idx] = data[strided_idx] * 2.0f;
    }
}

Memory Transaction Rules

For compute capability 6.0+, global memory transactions are 32 bytes. A warp of 32 threads accessing consecutive 4-byte floats results in a single 128-byte transaction (4 x 32-byte segments).

  • Aligned, consecutive access: 1 transaction per 32 threads
  • Misaligned access: up to 2 transactions per 32 threads
  • Strided access: up to 32 transactions per 32 threads (worst case)

Practical Guidelines

  1. Structure data for consecutive thread access patterns
  2. Use Structure of Arrays (SoA) instead of Array of Structures (AoS)
  3. Align data structures to 128-byte boundaries when possible
  4. Consider shared memory for irregular access patterns
  5. Profile with nsight compute to measure actual memory efficiency

Measuring Coalescing Efficiency

Use NVIDIA Nsight Compute to measure the gld_efficiency and gst_efficiency metrics. Values close to 100% indicate well-coalesced access patterns.
